## ðŸ› Common Issues & Troubleshooting

### If the app doesn't load:
- Check that the server is running: `pm2 list`
- Verify port 3000 is accessible: `curl http://localhost:3000/health`
- Restart the service: `pm2 restart mtn-edu-demo`

### If data doesn't persist:
- Demo uses localStorage - check browser console for errors
- Try "Reset Demo" to clear corrupted state
- Ensure browser allows localStorage

### If guided demo tooltip doesn't show:
- Toggle "Guided Demo" OFF then ON
- Tooltip appears based on current step

## ðŸ“ž Questions & Answers

**Q: Is this connected to a real database?**  
A: This demo uses browser localStorage. Production would integrate with Cloudflare D1 or enterprise databases.

**Q: Can I customize the solution library?**  
A: Yes, the library is defined in `data.js` and can be easily extended with new items and categories.

**Q: Does the AI recommendation really use machine learning?**  
A: The current version uses rules-based matching. Production could integrate OpenAI or custom ML models.

**Q: Can we integrate with Salesforce?**  
A: Not yet, but it's on the roadmap. The architecture supports API integration.

**Q: How do we handle real client data?**  
A: Production deployment would add authentication, database persistence, and POPIA-compliant data handling.
